Personal Finance Skill

In development · Anthropic Skill

Anthropic Skill that processes Brazilian bank statement PDFs through a stage-based state machine — parse, validate, classify merchants, and persist to a personal SQLite ledger.

Problem

Brazilian bank statements arrive as PDFs with inconsistent layouts (one per bank, one per statement type — credit card faturas vs. current-account extratos). Importing these by hand into a personal-finance ledger is a Saturday-morning ritual: parse, dedupe, classify each merchant. The cost compounds month over month.

Architecture

Anthropic Skill packaged as a stage-based state machine: parse PDF → normalize transactions → validate gates (running balance, sign conventions, duplicates) → classify each merchant against a learned table → persist to SQLite. Idempotent imports — re-running over the same PDF is a no-op, keyed on (bank, statement_id, transaction_hash). Merchant classification is self-improving: human-confirmed mappings are written back so subsequent runs need fewer interactive prompts.

Tech Stack

  • Anthropic Skills protocol
  • Python
  • SQLite
  • PDF parsing
  • state machine

Anthropic Skills protocol; Python; PDF parsing libraries; SQLite for the personal ledger; deterministic state machine with explicit validation gates between stages.

Engineering Highlights

  • Stage-based state machine with explicit validation gates — failed validation halts the pipeline at a known, debuggable boundary instead of silently corrupting the ledger.
  • Idempotent imports keyed on (bank, statement_id, transaction_hash) — running the Skill twice over the same PDF is a no-op.
  • Self-improving merchant classifier: confirmed mappings are written back to the ledger so the Skill needs fewer prompts on future runs.
  • Fits the Anthropic Skills protocol so the same workflow is callable from any Anthropic-Skills-aware client.

Outcomes

  • Personal-use Skill in active development.
  • Replaces a manual monthly bank-statement reconciliation with a single Skill invocation.
