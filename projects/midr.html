MIDR — Pavement Defect Detection

Production · Adopted

YOLO-ensemble defect detection deployed in highway inspection by the Court of Accounts of Ceará (TCE-CE). Custom IOU + NMS fusion across multiple class-specialized YOLO models.

Problem

Highway concessionaires and audit bodies need objective, scalable evidence of pavement condition. Manual inspection is slow, subjective, and inconsistent across reviewers. The system needs to detect potholes (buracos), cracks (trincas), patches (remendos), and signage from drive-by smartphone or vehicle-mounted footage — kilometers at a time.

Architecture

MIDR is an ensemble of YOLO models, each trained on a curated subset, orchestrated by a `Detector` class (`src/midr/dd.py`) that runs models in parallel, applies per-model IOU and NMS, then fuses detections via consensus, confidence, and class-priority rules. The training pipeline reads from per-version SQLite databases (`midr_18072024.db`, `midr_01052025_smartphone_hd.db`) and runs in Google Colab notebooks (`yolo_tl_2025.ipynb`) for reproducibility. End-to-end image and video processing scripts (`main_process_images.py`, `main_process_video.py`) take folders in `to_process/source` and write annotated outputs to `to_process/destination`.

Tech Stack

  • Python 3.11
  • PyTorch
  • Ultralytics YOLO 8.3.x
  • OpenCV
  • SQLite (dataset versioning)
  • Jupyter / Google Colab

Python 3.11; PyTorch + Ultralytics YOLO (8.3.3); OpenCV; SQLite-backed dataset versioning; Conda environment spec; Jupyter / Google Colab notebook for transfer-learning training.

Engineering Highlights

  • Custom IOU + NMS fusion logic combines predictions across the YOLO ensemble; consensus + confidence-weighted voting reduce false positives a single model would emit.
  • Per-version SQLite datasets make dataset evolution reproducible — the team retrains incrementally as new footage is collected.
  • Drop-in image and video processing scripts so field operators can process new footage without touching model code.
  • Class-specialized YOLO ensemble: each defect class (potholes, cracks, patches, signage) has its own training loop tuned for that class's failure modes.

Outcomes

  • Deployed as the standard inspection system used by the Court of Accounts of Ceará (TCE-CE).
  • Replaces the visual-inspection bottleneck for kilometers of recorded highway footage with an automated first-pass defect screen.
