CRIF — Calibrated Raster Interpolation

Open research

Geospatial Python library for unified grid rasterization, dasymetric mapping, and hotspot discovery. Built at NYU VIDA Center for the UrbanMapper project.

Problem

Urban analysts repeatedly hit the same wall: heterogeneous geospatial data — points, polygons, administrative units — needs to be aligned to a common raster grid for downstream Bayesian modeling and visualization. Each project ends up rewriting the same rasterization, normalization, and overlap-handling code, with subtle bugs that propagate into research findings and break reproducibility.

Architecture

CRIF (Calibrated Raster Interpolation) exposes a single `Crif` class that defines grid bounds and cell size, plus three primitives: `create_raster_from_point` (point aggregation, with optional min-max-scaled multi-band normalization), `polygon_to_raster` (polygon rasterization with `average` or `sum` overlap handling), and `dasymetric_mapping` (distributes a polygon-level statistic across a finer raster using prior weights). A `combine_rasters` helper supports hotspot discovery by summing or thresholding raster fusions across heterogeneous signals. Each primitive has a formal mathematical derivation in the README so research artifacts stay reproducible.

Python, GeoPandas, Rasterio, NumPy, custom Jupyter widgets, Docker. Editable install (`pip install -e .`) plus a Dockerfile so urban-science collaborators iterate without breaking environments.

Engineering Highlights

  • Formal mathematical derivation for every primitive (point rasterization, polygon rasterization with overlap handling, dasymetric mapping, multi-band normalization) lives alongside the code.
  • `combine_rasters` enables hotspot discovery — fusing e.g. robberies + violence rasters into a composite hotspot map.
  • Multi-band normalization (`create_raster_from_point_min_max_scaled`) so heterogeneous priors can be combined on a common scale.
  • Custom Jupyter widgets for interactive raster exploration in research notebooks.
  • Editable install plus Dockerfile so collaborators reproduce results without environment churn.

Outcomes

  • Adopted internally by NYU VIDA Center's UrbanMapper project for urban Big Data analytics.
  • Reduces the boilerplate cost of rasterizing and normalizing new datasets from days of project setup to a few `Crif` calls.
